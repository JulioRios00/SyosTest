## ğŸ¯ Design Principles

### 1. Dependency Inversion
- Domain layer has **no dependencies** on outer layers
- Outer layers depend on domain interfaces (ports)
- Business logic is independent of frameworks and databases

### 2. Separation of Concerns
- **Domain**: Business rules and entities
- **Application**: Use cases and orchestration
- **Infrastructure**: Technical implementations
- **Adapters**: External interfaces (HTTP, messaging)

### 3. Testability
- Each layer can be tested independently
- Easy to mock dependencies using interfaces
- Business logic can be tested without infrastructure
